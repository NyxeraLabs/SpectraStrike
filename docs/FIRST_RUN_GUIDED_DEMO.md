# SpectraStrike + VectorVue First-Run Guided Demo (Detailed)

## Purpose
This guide documents two complete onboarding modes for local federation:
1. Guided tour with pre-seeded multi-tenant demo data.
2. Clean first-run assisted mode with reset state so operators configure their own environment.

The guide covers both Web UI and TUI workflows.

## Prerequisites
1. Docker + Docker Compose installed.
2. Both repositories available side-by-side:
- `/home/xoce/Workspace/SpectraStrike`
- `/home/xoce/Workspace/VectorVue`
3. TLS certificates generated by each project defaults.

## Mode A: Guided Tour With Seeded Data

### Step 1. Start local federation stacks
1. In `SpectraStrike`:
```bash
cd /home/xoce/Workspace/SpectraStrike
make local-federation-up
```

### Step 2. Seed both tenants (ACME + Globex)
1. In `SpectraStrike`:
```bash
make demo-seed
```
This action seeds:
- VectorVue PostgreSQL tenant/campaign/client datasets (ACME + Globex).
- SpectraStrike runtime playbooks + task queue for both tenant IDs.

### Step 3. Follow guided web flow
1. Open SpectraStrike workflow:
- `https://127.0.0.1:18443/ui/dashboard/workflow`
2. Use `Next Demo Step` in `Assisted First-Run Demo` panel.
3. Move to Nexus from demo CTA.
4. Move to VectorVue validation from Nexus CTA.

### Step 4. Validate telemetry and graph surfaces
1. SpectraStrike Telemetry:
- `https://127.0.0.1:18443/ui/dashboard/telemetry`
2. SpectraStrike ASM:
- `https://127.0.0.1:18443/ui/dashboard/asm`
3. VectorVue Analytics:
- `https://127.0.0.1/portal/analytics`

### Step 5. Guided TUI walkthrough (optional)
1. SpectraStrike TUI:
```bash
cd /home/xoce/Workspace/SpectraStrike
make open-tui
```
2. VectorVue TUI:
```bash
cd /home/xoce/Workspace/VectorVue
make run-tui
```

## Mode B: Clean First-Run Assisted Setup

### Step 1. Reset both sides to zero
1. In `SpectraStrike`:
```bash
cd /home/xoce/Workspace/SpectraStrike
make demo-reset
```
This action resets:
- SpectraStrike PostgreSQL database.
- SpectraStrike in-memory demo runtime stores.
- VectorVue PostgreSQL public schema.

### Step 2. Bring stacks up again
1. In `SpectraStrike`:
```bash
make local-federation-up
```

### Step 3. Start clean assisted run
1. Open SpectraStrike workflow:
- `https://127.0.0.1:18443/ui/dashboard/workflow`
2. Complete assisted first-run steps with empty baseline.
3. Create your own campaigns, tasks, and policies.

### Step 4. Configure from TUI (clean state)
1. SpectraStrike TUI:
```bash
cd /home/xoce/Workspace/SpectraStrike
make open-tui
```
2. VectorVue TUI:
```bash
cd /home/xoce/Workspace/VectorVue
make run-tui
```

## Recommended Day-0 Operator Sequence
1. `make local-federation-up` (SpectraStrike).
2. `make demo-seed` for walkthrough training.
3. Practice full UI flow SpectraStrike -> Nexus -> VectorVue.
4. `make demo-reset`.
5. Start clean assisted first-run and configure production-like settings.

## Notes
- Cross-app links use HTTPS-only resolution.
- If browser still shows stale UI, rebuild UI images:
- SpectraStrike: `make ui-web-rebuild`
- VectorVue: `make -C ../VectorVue portal-rebuild-ui`
